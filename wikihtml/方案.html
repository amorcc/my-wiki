橡木桶进销存系统方案

一、名词定义及相关情况说明 1、橡木桶进销存系统，以下简称“系统”。 2、本系统针对的对象：进行葡萄酒商品销售的店铺。 3、开发人员简介（本人06年获得国家信息产业厅颁发的“高级程序员”证书，从事软件开发行业7年，从业第二年开始做软件开发管理工作，职位为技术总监。从业期间做过北京地铁媒体管理系统、广州地铁媒体管理系统、河南公安厅反恐系统、河南锅炉管理系统等企事业单位管理系统。同时，互联网方向做过关于幼儿学前阶段的互联网项目。项目涉及PC网站、winform桌面程序、wpf动画程序、andriod应用、ios应用等。

二、软件开发及运行环境

1、软件开发环境：

vs 2010 c# winform + mysql数据库 根据实际项目使用环境，我们使用winform+mysql开发本套系统，可以方便软件的操作，同时可以更好的兼容条形码扫描枪和条码打印机等硬件设备。Mysql数据库为开源数据库，不需要重复投资，占用系统硬件资源更好，让软件和电脑运行更加流程。

2、软件运行环境

windows 7 旗舰版及以上操作系统。推荐使用windows 7 旗舰版。 所需软件框架：.net framework 4.0及以上。

三、软件特点

1、建立详细葡萄酒商品数据库，通过EXCEL制式表格快速完善葡萄酒商品数据库。保存葡萄酒英文名称、中文名称、产地、生产年份、口味特点、葡萄酒图片等信息，为系统其他功能提供数据支持。

2、自动生成葡萄酒商品内部条形码，对于已经有商品条形码的，直接记录条码信息到数据库，没有商品条形码的自动生成内部条形码，并可打印。

3、客户关系管理模块与商品销售价格挂钩，可以创建多种销售价格打折方案，客户购买时自动计算折扣信息。客户购买后，记录客户购买商品的记录，为销售策略制定提供数据支持。

4、完善的软件权限，根据员工职位不同，设置多种操作权限，保障系统数据的安全。

四、本版本软件缺陷

因系统本版本是单机版，只能实现单个局域网的信息互通，因此会形成单个店铺的信息孤岛，信息统计无法统计总体产品销售情况，信息分析的结果会具有片面性。建议在本版本开发完成后，进行网络版开发，实现所有店铺的信息统一管理，对所有店铺的销售信息、客户信息、库存信息全面统计分析，实现大数据的应用，实现多店铺之间的库存调配等。

五、开发模型选择及开发周期

本版本软件使用原型开发法和螺旋模型开发法相结合的开发模式，先将核心业务使用原型开发法，快速建立软件的基础原型，搭好软件结构框架。然后进入正式代码编写过程，通过螺旋模型开发法，逐步递归迭代软件产品，减少因开发人员因需求和业务的不熟悉而造成的返工现象。 本版本软件开发大概需要2个月，2个月后进入试运行阶段，试运行阶段由开发人员和具体业务人员共同组成测试小组，完善软件流程和软件的功能。在2个月的开发周期内，建立产品开发关键点和关键任务，定时向客户反馈软件开发信息。

七、软件模块及主要功能

1、葡萄酒商品信息库模块 主要完善葡萄酒的商品信息库，葡萄酒的基础信息（如:中文名称、英文名称、净容量、酒精度、产地、生产日期等），葡萄酒的品鉴信息（如：口味特点、酸甜度特点、生产材料特点、储藏条件等），葡萄酒的背景资料信息（如：葡萄酒的产地介绍、葡萄酒生产商介绍、葡萄酒的历史介绍、葡萄酒宣传视频等），葡萄酒的电子识别信息（如：商品条形码信息等） 主要功能： 1.1 新增葡萄酒商品信息 1.2 修改、删除葡萄酒商品信息 1.3 葡萄酒背景资料文件的上传下载 1.4 自动生成葡萄酒内部条形码，打印葡萄酒背标。 1.5 商品信息查询功能，通过模糊查询，可以根据产地、名称、口味、生产商、条形码等信息，进行模糊查询，展示商品详细信息。

2、库存管理模块 主要实现店铺的葡萄酒库存管理，入库时，根据商品外包装箱的具体情况，可以选择打印外包装箱的商品标签，方便客户管理和盘存商品，对现有所有商品的库存信息预警等功能。 主要功能： 1.1 商品入库 1.2 商品外包装箱、商品背标信息打印 1.3 商品库存信息查询 1.4 商品外包装箱信息管理 1.5 商品库存信息预警（设定商品的库存预警值，通过该值和历史同期销售额情况，对库存进行预警） 1.6 年、季度、月、日销售情况快速盘存

3、销售管理模块 主要实现商品的定价策略制定，商品销售记录，销售信息的简要统计等功能。 主要功能： 1.1 商品定价策略制定（根据会员卡级别进行商品价格自动浮动，根据客户信息进行商品价格自动浮动，根据人员权限限制销售人员销售某一产品的价格区间等） 1.2 销售记录（店铺吧台扫描商品条形码，输入购买客户信息，输入商品数量，自动计算商品单价总价，并更新库存，记录销售人员信息。注：可直接扫描单品或者外包装箱条形码，系统自动计算包装箱产品种类及数量） 1.3销售情况日报和周报（实现日、周销售简报，可以直接复制通过QQ、邮件等方式向店主店长等管理人员发送。）

4、客户关系管理模块 主要实现客户基础信息、客户会员信息、客户资料的管理，以及客户购买产品的统计等功能，进行权限管理，确保客户信息安全。 1.1 客户基础信息管理（客户姓名、联系电话、住址、单位、职务、爱好等基础信息管理） 1.2 客户会员信息管理（客户会员级别、客户会员等级升降等） 1.3 客户资料管理（客户身份证、客户单位相关资料等，可以直接上传文件到系统）可选 1.4 客户购买产品统计（统计客户购买产品的总价格、种类。分析客户时间段内的购买曲线） 1.5 客户回访记录管理（销售人员回访客户后，记录客户回访简报）

5、统计分析模块 5.1 店铺库存信息统计 5.2 店铺销售情况统计 5.3 店铺上架商品信息统计 5.4 客户消费情况统计 5.5 客户回访情况统计 5.6 根据会员级别统计销售产品类别及数量 5.7 统计销售人员销售业绩

6、软件用户及权限管理模块 主要实现店铺员工基础信息管理，创建工作人员的用户名密码，分配工作人员的权限等。 6.1 店铺员工信息管理（员工姓名、性别、职位、籍贯、详细地址、联系电话、QQ号码、EMAIL、身份证扫描文件等） 6.2 员工用户名管理（创建员工登录用户名和初始密码，还原初始密码等） 6.3 员工权限管理（根据工种和职位，分配软件功能的操作权限给员工)

7、系统设置模块 7.1 系统字典表定义（店铺员工职位字典表定义、店铺会员级别字典表定义等） 7.2 店铺基础信息管理（店铺名称、店铺地址、联系电话、负责人、主营产品等信息，并将信息编码到二维码中，客户可以通过手机任意二维码扫描软件，快速记录店铺的相关信息） 7.3 修改用户名密码（修改当前登录用户的用户名密码）

8、管理员模块 系统将创建超级管理员用户，该用户具有软件的所有权限 8.1 管理员用户名密码修改 8.2 系统数据库备份 8.3 客户信息备份
